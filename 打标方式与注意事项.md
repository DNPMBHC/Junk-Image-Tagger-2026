# 打标方式与注意事项（2026-02-24）

本文档用于说明当前前端版本的推荐使用顺序、打标流程与关键注意事项。

## 1. 导航读取顺序（从前到后）

一级界面顺序：

1. 图片打标
2. 打标补全
3. 文件目录管理
4. Tag 管理
5. 打标日志
6. 其他设置（默认折叠）

“其他设置”展开后的二级菜单顺序：

1. 渠道管理
2. 调度组-渠道规则
3. 提示词预设
4. 配置中心

## 2. 新版打标方式（推荐流程）

### 步骤 1：先做基础配置

1. 在“渠道管理”配置 API URL、API Key，并拉取模型测试连通性。
2. 在“调度组-渠道规则”创建调度组，配置步骤、超时、重试和间隔。
3. 在“文件目录管理”维护分组与本地目录映射。

### 步骤 2：执行图片打标

1. 进入“图片打标”，选择分组与调度组。
2. 按需套用系统/用户提示词预设。
3. 启动打标任务，先观察结果列表与日志状态。
4. 结果确认后再执行批量保存。

### 步骤 3：执行打标补全

1. 进入“打标补全”，选择同一分组与调度组。
2. 仅针对不达标文本执行补全。
3. 抽样检查补全结果后再保存。

### 步骤 4：做 Tag 精修

1. 在“Tag 管理”进行单条编辑、去重、替换。
2. 保存后再回看“打标日志”确认是否有失败项。

## 3. 新手提示符号说明

页面中提供 `⚪ + ❓` 形式的提示入口。点击后会展开当前面板注释，包含：

1. 当前面板的用途
2. 推荐操作顺序
3. 关联后端接口（用于排查）

建议首次使用时，先逐个面板查看提示再执行任务。

## 4. 关键默认值与配置建议

“调度组-渠道规则”中的全局字符数建议值：

1. 最小字符数：`50`
2. 最大字符数：`2048`

建议：

1. 每次升级后先打开该面板确认并保存一次规则，避免历史数据覆盖新默认值。
2. 先用小批量样本验证通过后，再执行大批量任务。

## 5. 主题与字体配置

“配置中心”中在“配置诊断”下方提供“颜色与字体自定义”，支持：

1. 主色、辅色、背景、侧栏、字体颜色调整
2. 全局字体族、字号、标题字号、行高调整
3. 快捷预设一键切换

建议：

1. 先应用预设，再做微调。
2. 调整后点击“保存主题”，避免刷新丢失。

## 6. 运行与数据安全注意事项

1. 启动前确认端口未被占用（默认 `30101`）。
2. 若页面无法打开，先检查 `http://127.0.0.1:30101/api/health` 是否返回 `{"ok":true}`。
3. 执行“清空配置”前先在配置中心导出备份。
4. 运行数据以 `data.json` 为准，建议定期备份。
5. 打包版运行推荐使用 `dist/start-console.bat`，便于观察命令行日志。

## 7. 常见排查顺序

1. 看“打标日志”定位失败阶段（读图、调度、保存）。
2. 在“渠道管理”检查 API 连通性与模型可用性。
3. 在“调度组-渠道规则”检查字符阈值、重试与超时设置。
4. 在“配置中心 > 配置诊断”查看异常项并修复。
